    etcd Operations Standard¶

    Purpose¶

    This standard describes public-safe etcd operations for Kubernetes and distributed systems.

    Operating Principles¶

    • Treat etcd as critical state.
    • Use an odd number of members for quorum.
    • Back up before cluster changes.
    • Monitor leader changes, disk latency, database size, and alarms.
    • Protect peer and client certificates.

    Backup¶

    ETCDCTL_API=3 etcdctl snapshot save snapshot.db

    Store snapshots encrypted and test restore regularly.

    Health Checks¶

    ETCDCTL_API=3 etcdctl endpoint health
ETCDCTL_API=3 etcdctl endpoint status --write-out=table

    Kubernetes Notes¶

    For managed Kubernetes, control plane etcd is usually provider-managed. For self-managed clusters, etcd backup and restore must be part of the platform runbook.

    Publication Safety¶

    Do not publish real etcd certificates, keys, endpoints, snapshots, or cluster member names.

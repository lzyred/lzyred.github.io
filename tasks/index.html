    按任务进入¶

    先选择当前要完成的任务，再按文档类型或技术领域缩小范围。页面只呈现当前任务，不再要求用户纵向浏览六组相同的大列表。

    文档类型
    技术领域

    01 · ARCHITECTURE

    设计生产架构

    在实施前确定可用性、容量、安全、依赖、升级和恢复边界。

    5 ASSETS
    • 设计 AWS EKS 高可用生产集群

      形成可交付的集群架构与运行边界。

      Standard平台
    • 建立 Kubernetes 集群供给标准

      定义有顺序的平台依赖图与交付门禁。

      Standard平台
    • 确定 Redis 生产运行最低要求

      建立内存、持久化、高可用与恢复基线。

      Standard数据
    • 建立 Linux 服务器基线

      形成可验证的主机安全与运行基线。

      Standard平台
    • 设计 Nginx 反向代理与 TLS

      定义入口、证书、安全和故障边界。

      Standard平台
    02 · DELIVERY

    部署与上线

    按前置条件、执行顺序、验证和停止条件完成可靠交付。

    5 ASSETS
    • 使用 kubeadm 交付 Kubernetes

      覆盖控制面、etcd、证书、插件顺序与重建边界。

      Standard平台
    • 使用 Docker Compose 交付服务栈

      覆盖运行契约、依赖、数据和验证。

      Standard交付
    • 建设 Harbor 镜像仓库

      覆盖存储、TLS、权限、备份和恢复。

      Standard平台
    • 建立 Argo CD GitOps 交付

      覆盖事实源、同步、权限和回滚控制。

      Standard交付
    • 建立 Loki 日志平台

      覆盖采集、标签、留存、查询和告警。

      Standard可观测性
    03 · CONTROLLED CHANGE

    变更与升级

    把升级、迁移、证书轮换和回滚作为受控变更，而不是孤立命令。

    5 ASSETS
    • 判断回滚是否真的成立

      判断数据兼容、外部行为和自动化事实源。

      Concept交付
    • 执行 Kubernetes Deployment 回滚

      包含影响判断、执行、验证、停止和升级条件。

      Runbook交付
    • 规划 EKS 与 Kubernetes 升级

      覆盖 API、插件、节点和可逆边界。

      Standard平台
    • 规划主机应用容器化迁移

      从旧主机中识别并重建运行契约。

      Case交付
    • 规划对象存储迁移与写入切换

      将切换视为写入权威变更，而不是复制完成。

      Case平台
    04 · SIGNALS

    监控与排障

    建立信号与告警，再沿应用、采集、传输和后端逐跳验证。

    5 ASSETS
    • 建立 Prometheus 告警标准

      定义信号质量、路由、抑制、响应和证据。

      Standard可观测性
    • 接入 Prometheus Exporter

      定义部署位置、权限、down 区分和必备告警。

      Standard可观测性
    • 理解 OpenTelemetry 组件与数据流

      明确信号、上下文、Collector 和后端边界。

      Concept可观测性
    • 灰度接入 OpenTelemetry 自动埋点

      使用独立 canary、分层验证和显式回滚。

      Runbook可观测性
    • 排查遥测数据缺失

      沿应用、注入、协议、Collector 和后端逐跳定位。

      Runbook可观测性
    05 · INCIDENT RESPONSE

    恢复与应急

    保全证据，确认影响，执行受控恢复，并完成验证、升级和关闭。

    5 ASSETS
    • 检查并恢复 Redis AOF

      覆盖证据保全、隔离验证、修复和恢复。

      Runbook数据
    • 执行 Redis 迁移与故障切换

      覆盖角色确认、数据验证、切换和回退。

      Runbook数据
    • 处理 Kubernetes PVC 使用率告警

      判断真实使用量、风险、扩容路径和清理边界。

      Runbook平台
    • 扩展 Linux 磁盘与 LVM

      覆盖设备、分区、PV、VG、LV 和文件系统验证。

      Runbook平台
    • 处理 TLS 证书到期

      覆盖影响确认、替换、验证和回滚。

      Runbook平台
    06 · GOVERNANCE

    文档与评审

    将技术意图、执行步骤、验证、失败模式和回滚写成可审查结果。

    5 ASSETS
    • 编写迁移计划

      统一范围、依赖、切换、验证、回退和沟通。

      Template工程效能
    • 编写生产 Runbook

      统一前置、执行、验证、停止、回滚和升级条件。

      Template工程效能
    • 编写生产标准

      统一目的、架构、要求、例外、验证和复审。

      Template工程效能
    • 编写事故沟通与调查材料

      将事实、影响、证据和判断分离。

      Concept工程效能
    • 设计可执行的工程交接

      明确目标、状态、证据、风险和下一步。

    Nginx Reverse Proxy and TLS Standard¶

    Purpose¶

    This standard defines safe public patterns for deploying Nginx or OpenResty as a reverse proxy with TLS.

    Core Principles¶

    1. Separate public listener configuration from upstream service configuration.
    2. Use TLS certificates from a controlled lifecycle process.
    3. Do not publish real certificates, private keys, domains, or upstream addresses.
    4. Use health checks and reload validation before applying changes.
    5. Keep access logs useful while avoiding sensitive request data where possible.

    Basic Reverse Proxy Template¶

    server {
    listen 80;
    server_name service.example.com;

    location / {
        proxy_pass http://example-upstream;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 5s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}

    Upstream example:

    upstream example-upstream {
    server 10.0.0.10:8080 max_fails=3 fail_timeout=30s;
    server 10.0.0.11:8080 max_fails=3 fail_timeout=30s;
}

    TLS Template¶

    server {
    listen 443 ssl http2;
    server_name service.example.com;

    ssl_certificate /etc/nginx/certs/service.example.com.crt;
    ssl_certificate_key /etc/nginx/certs/service.example.com.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;

    location / {
        proxy_pass http://example-upstream;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }
}

    The key path is illustrative. Never commit actual private key files.

    Certificate Expiry Check¶

    echo | openssl s_client -servername service.example.com -connect service.example.com:443 2>/dev/null \
  | openssl x509 -noout -dates

    For local files:

    openssl x509 -in /etc/nginx/certs/service.example.com.crt -noout -dates

    Reload Procedure¶

    sudo nginx -t
sudo systemctl reload nginx
sudo systemctl status nginx --no-pager

    Validation:

    curl -I https://service.example.com
curl -sS https://service.example.com/health

    Common Failure Modes¶

    Symptom Likely Cause Check
    502 Bad Gateway Upstream unavailable or wrong port curl upstream from proxy host
    TLS warning Expired or wrong certificate openssl x509 -noout -dates
    Redirect loop Incorrect forwarded protocol Check X-Forwarded-Proto handling
    Large upload fails Body size too small client_max_body_size
    Long request times out Proxy timeout too low proxy_read_timeout

    Security Notes¶

    • Store private keys with restrictive permissions.
    • Use automation for renewal where possible.
    • Test renewal in staging before production.
    • Avoid logging authorization headers.
    • Do not expose admin routes through public listeners unless explicitly required and protected.
